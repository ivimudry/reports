active_player_period — атрибут Customer.io

Что это: период суток, когда игрок чаще всего заходит на сайт. Используется маркетингом для отправки рассылок в активное время игрока.

Атрибут

ИмяТипЗначения
active_player_periodstringсм. таблицу ниже или unknown

Периоды (локальное время игрока)

ЗначениеЧасы
late_night00:00–03:59
early_morning04:00–07:59
morning08:00–11:59
afternoon12:00–15:59
evening16:00–19:59
night20:00–23:59
unknownвходов мало или нет таймзоны

Алгоритм

  1. Берём последние 30 входов игрока на сайт (поле session.start_time, UTC).
  2. Если входов < 5 или у игрока нет таймзоны → unknown, выходим.
  3. Каждый start_time переводим в локальное время игрока, берём только час (0–23).
  4. Каждый вход даёт очки бакетам:
    • +1.0 в свой бакет;
    • +0.5 в предыдущий бакет;
    • +0.5 в следующий бакет.
    Бакеты идут по кругу: после night снова late_night.
  5. Победитель — бакет с наибольшей суммой очков. Это и есть active_player_period.
  6. Записать в Customer.io через API.
Зачем «+0.5 соседям»: вход в 15:55 формально попадает в afternoon, но фактически он почти в evening. Соседи получают половину очка, чтобы границы бакетов не били результат. Если у игрока есть один чёткий период — он всё равно победит. Если активность размазана между двумя соседями — победит тот, к которому игрок ближе.
Важно: бакеты замкнуты в круг. Сосед late_night слева — это night, сосед night справа — late_night. Иначе игрок, заходящий в 23:00 и 01:00, получит ерунду.

Маппинг час → бакет

bucket_index = floor(hour / 4)
0 → late_night
1 → early_morning
2 → morning
3 → afternoon
4 → evening
5 → night

Python

BUCKETS = ['late_night', 'early_morning', 'morning',
           'afternoon', 'evening', 'night']

def active_period(hours):
    if len(hours) < 5:
        return 'unknown'
    score = [0.0] * 6
    for h in hours:
        i = h // 4
        score[i] += 1.0
        score[(i - 1) % 6] += 0.5
        score[(i + 1) % 6] += 0.5
    return BUCKETS[score.index(max(score))]

# пример: входы преимущественно вечером
print(active_period([16, 17, 18, 19, 20, 15, 19, 21]))  # evening

SQL (PostgreSQL)

WITH buckets AS (
  SELECT
    user_id,
    FLOOR(EXTRACT(HOUR FROM start_time AT TIME ZONE user_tz) / 4)::INT AS b
  FROM sessions
  WHERE start_time > NOW() - INTERVAL '30 days'
),
counts AS (
  SELECT user_id, b, COUNT(*) AS c
  FROM buckets
  GROUP BY user_id, b
),
-- разносим очки: +1 в свой бакет, +0.5 в соседей (по кругу)
scored AS (
  SELECT user_id, b AS bucket, c::FLOAT AS score FROM counts
  UNION ALL
  SELECT user_id, (b + 1) % 6, c * 0.5 FROM counts
  UNION ALL
  SELECT user_id, (b + 5) % 6, c * 0.5 FROM counts  -- (b - 1 + 6) mod 6
),
totals AS (
  SELECT user_id, bucket, SUM(score) AS total,
         ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY SUM(score) DESC) AS rn
  FROM scored
  GROUP BY user_id, bucket
),
qualified AS (
  SELECT user_id FROM buckets GROUP BY user_id HAVING COUNT(*) >= 5
)
SELECT
  q.user_id,
  CASE t.bucket
    WHEN 0 THEN 'late_night'
    WHEN 1 THEN 'early_morning'
    WHEN 2 THEN 'morning'
    WHEN 3 THEN 'afternoon'
    WHEN 4 THEN 'evening'
    WHEN 5 THEN 'night'
  END AS active_player_period
FROM qualified q
JOIN totals t ON t.user_id = q.user_id AND t.rn = 1;

Расписание

Батч-джоб раз в сутки в 03:00 UTC. Считает по всем активным игрокам, пушит результат в Customer.io через API.
