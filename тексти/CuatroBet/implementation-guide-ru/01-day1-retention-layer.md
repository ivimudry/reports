# 01 — Day 1 Retention Layer — Руководство по внедрению

**Цепочка:** Day 1 Retention Layer  
**Приоритет:** Наивысший  
**Ожидаемое количество коммуникаций:** 7 (путь Slots) + 7 (путь Aviator) = 14 всего  
**Пререквизиты из мастер-документа:** События `deposit_success`, `session_start`, `session_end`, `game_launched`; Атрибуты `game_preference`, `deposit_count`, `ftd_date`; Сегменты `SEG-FTD`, `SEG-D1LAYER`, `SEG-GAME-SLOTS`, `SEG-GAME-AVIATOR`

---

## ШАГ 0: ПРЕДСТАРТОВЫЙ ЧЕК-ЛИСТ

Перед созданием любых кампаний убедитесь:

- [ ] Событие `deposit_success` срабатывает в реальном времени (≤5 секунд) с флагом `is_ftd`
- [ ] События `session_start` / `session_end` срабатывают корректно
- [ ] Событие `game_launched` срабатывает с полем `game_type`
- [ ] Атрибут `game_preference` заполнен (используется для ветвления путей)
- [ ] Бонусный шаблон `WBL-S-D1` существует (50 FS на Gates of Olympus, отыгрыш 20×, срок действия 24ч)
- [ ] Бонусный шаблон для Aviator: 5 Free Flights, срок действия 24ч
- [ ] Deep links настроены: `{base_url}/game/gates-of-olympus`, `{base_url}/game/aviator`, `{base_url}/cashier`
- [ ] Система In-app всплывающих окон активна с возможностью триггера менее чем за секунду
- [ ] SMS-шлюз активен, идентификатор отправителя = "CuatroBet"
- [ ] App Push активирован (опционально, но рекомендуется — без него теряется ~30% ветки T+6ч)

---

## ШАГ 1: СОЗДАНИЕ JOURNEY

**Название кампании:** `01-D1RET-JOURNEY`  
**Тип:** Triggered journey (на основе события)  
**Триггер входа:** Событие = `deposit_success` WHERE `is_ftd = true`  
**Повторный вход:** Нет (один раз на игрока)  
**Окно journey:** 72 часа с момента входа  
**Условия выхода:**
- Игрок достигает `deposit_count >= 2` в любой момент → выход в Post-FTD cadence
- Прошло 72 часа → выход в Reactivation Ladder (если не вернулся) или Post-FTD (если вернулся)

---

## ШАГ 2: ДОБАВЛЕНИЕ ВЕТВЛЕНИЯ — Slots vs Aviator

**Позиция:** Сразу после триггера входа  
**Тип ветвления:** Условное разделение  
**Условие:**
- **Путь Aviator:** `game_preference = "aviator"` ИЛИ тип игры первой сессии = `crash`
- **Путь Slots:** Всё остальное (по умолчанию)

Ожидаемое распределение: ~95% Slots / ~5% Aviator

### 3.1 Касание 1 — Мгновенное начисление FTD (системное действие)

| Настройка | Значение |
|-----------|----------|
| **Campaign ID** | `01-D1RET-T1-SYSTEM` |
| **Тип** | Системное действие (сообщение не отправляется) |
| **Время** | T+0 (сразу после FTD) |
| **Действие** | Вызов Bonus API → Начислить бонус `WBL-S-D1` |

**Шаги настройки:**
1. Добавить узел "Action" сразу после ветки Slots
2. Тип действия: "Grant Bonus"
3. Bonus ID: `WBL-S-D1` (50 фриспинов без депозита)
4. Игра: Gates of Olympus (по умолчанию) ИЛИ игра первой сессии игрока, если другая
5. Срок действия: 24 часа с момента начисления
6. Отыгрыш: 20× на выигрыш
7. Сообщение не отправляется — это скрытое системное действие

## ШАГ 5: ЧЕК-ЛИСТ ТЕСТИРОВАНИЯ

- [ ] FTD срабатывает и journey запускается в течение 5 секунд
- [ ] Ветвление корректно определяет игроков Aviator vs Slots
- [ ] Бонус Касания 1 зачисляется мгновенно (проверить бонусный баланс игрока)
- [ ] Поп-ап Касания 2 появляется точно на T+5мин для игроков в сессии
- [ ] Поп-ап Касания 2 НЕ появляется для игроков, ушедших до T+5мин
- [ ] Точка решения T+30мин корректно определяет "в сессии" vs "ушёл"
- [ ] SMS Касания 4 приходит на T+2ч для игроков, покинувших сессию
- [ ] Точка решения T+6ч корректно определяет возврат vs отсутствие возврата
- [ ] Поп-ап Касания 5A + зачисление 100 CC для вернувшихся игроков
- [ ] SMS Касания 5B + временный бонус 1,000 ARS для невернувшихся
- [ ] Email Касания 6 отправляется только неактивным 24ч игрокам
- [ ] Бонус Касания 6 (75% match + 20 FS) автоматически зачисляется при депозите
- [ ] SMS Касания 7 отправляется только неактивным 48ч игрокам
- [ ] Бонус Касания 7 (500 ARS free bet) автоматически зачисляется
- [ ] Journey корректно завершается на 72ч
- [ ] Лимиты частоты соблюдены (макс. 3 сообщения/день в рамках D1 Layer)
- [ ] UTM-параметры присутствуют во всех ссылках
