# 06 — Неудачные депозиты: Коммуникационные лестницы — Руководство по внедрению

**Цепочка:** Неудачные депозиты (пост-восстановительное сопровождение)  
**Начальная точка:** T+5 мин после неудачного депозита (цепочка 04 обрабатывает T+0 до T+5мин)  
**Охват:** 5 причин отказа × 4 коммуникации = 20 коммуникаций всего  
**Предварительные требования из мастер-документа:** События `deposit_failed`, `cashier_opened`, `cashier_closed`, `deposit_success`; Атрибуты `deposit_count`, `welcome_bonus_claimed`; Сегменты `SEG-CASHIER-ABANDONED`

---

## ШАГ 0: ПРЕДПОЛЁТНЫЙ ЧЕК-ЛИСТ

- [ ] Цепочка 04 (Payment Failure Recovery) запущена и передаёт управление на T+30мин
- [ ] Событие `cashier_closed` срабатывает, когда игрок покидает страницу депозита без совершения депозита
- [ ] Коды причин отказа доступны: `INSUFF`, `METHOD`, `DECLINE`, `TIMEOUT`, `FRAUD`
- [ ] Атрибут `attempt_count_session` игрока отслеживает количество повторных попыток
- [ ] API тикетов поддержки доступен (для причины 3 и причины 5)
- [ ] Очередь проверки на мошенничество существует (для причины 5)
- [ ] Все бонусные шаблоны из мастер-документа созданы (приветственный бонус, компенсационные кредиты)

---

## ШАГ 1: СОЗДАТЬ 5 ОТДЕЛЬНЫХ JOURNEY

Создайте по одному journey на каждую причину отказа. Это сохраняет логику чистой и позволяет независимую оптимизацию.

| Название Journey | Триггер входа | Время старта |
|-----------------|---------------|--------------|
| `06-FAIL-R1-JOURNEY` | `cashier_closed` + нет попытки депозита + 30 мин неактивности | T+30 мин |
| `06-FAIL-R2-JOURNEY` | `deposit_failed` + `failure_reason = TIMEOUT/METHOD` + attempt_count = 1 + не восстановлен в цепочке 04 | T+5 мин |
| `06-FAIL-R3-JOURNEY` | `deposit_failed` + `attempt_count_session >= 2` + техническая причина + не восстановлен | T+5 мин |
| `06-FAIL-R4-JOURNEY` | `deposit_failed` + `failure_reason = DECLINE/INSUFF` + не восстановлен | T+5 мин |
| `06-FAIL-R5-JOURNEY` | `deposit_failed` + `failure_reason = FRAUD` | T+5 мин |

**Глобальный выход для всех journey:** событие `deposit_success` → немедленный выход из journey.

## ШАГ 7: ЧЕК-ЛИСТ ТЕСТИРОВАНИЯ

- [ ] Каждая причина корректно направляет в свой journey
- [ ] Глобальный выход срабатывает по событию `deposit_success` (немедленно останавливает все коммуникации)
- [ ] Причина 1: Закрытие кассы + 30 мин неактивности запускает journey
- [ ] Причина 2: Одиночный технический сбой + нет восстановления → journey стартует на T+5мин
- [ ] Причина 3: 2+ сбоя обнаружено → тикет поддержки создан автоматически
- [ ] Причина 3: Предпочтительный канал связи игрока записан и используется
- [ ] Причина 4: Альтернативные способы оплаты отображаются динамически
- [ ] Причина 5: Аккаунт помечен → очередь проверки на мошенничество
- [ ] Причина 5: Касания 3–4 срабатывают только при ложных срабатываниях
- [ ] Кредиты-извинения (4,100 ARS / 10,500 ARS) зачислены корректно
- [ ] Количество символов SMS ≤160 для всех сообщений
- [ ] UTM-параметры на всех ссылках
- [ ] Частотные ограничения соблюдены (макс. 2/день для pre-FTD)
- [ ] Нет пересечения с цепочкой 04 (PFR обрабатывает только T+0 до T+5мин)
