# 04 — Восстановление после ошибки платежа — Гайд по внедрению

**Цепочка:** Восстановление после ошибки платежа  
**Приоритет:** Высокий (самая дешёвая инициатива, наивысший ROI)  
**Ожидаемые коммуникации:** 4 маршрута ошибок + 1 празднование = 5 касаний  
**Требование к скорости:** Все попапы восстановления должны срабатывать в течение 5 СЕКУНД после события ошибки  
**Предварительные требования из Master Doc:** Событие `deposit_failed` с полем `failure_reason` (`INSUFF`/`METHOD`/`DECLINE`/`TIMEOUT`); API платёжных методов; Deep links `{base_url}/cashier?method={method}`, `{base_url}/cashier?amount={amount}`

---

## ШАГ 0: ПРЕДВАРИТЕЛЬНЫЙ ЧЕК-ЛИСТ

- [ ] Событие `deposit_failed` срабатывает в течение 5 секунд с точными кодами `failure_reason`
- [ ] Коды причин ошибок различаются: `INSUFF`, `METHOD`, `DECLINE`, `TIMEOUT`
- [ ] Система InApp-попапов может срабатывать в течение 5 секунд после события (требуется субсекундная задержка)
- [ ] API платёжных методов доступен (для отображения альтернативных методов)
- [ ] Deep links поддерживают предвыбранный метод оплаты: `{base_url}/cashier?method={method_id}`
- [ ] Deep links поддерживают предзаполненную сумму: `{base_url}/cashier?amount={amount}`
- [ ] Доступен механизм автоматического повтора для ошибок по таймауту
- [ ] Шаблон бонуса 100 CC (Cuatro Coins) готов для празднования восстановления
- [ ] Передача в цепочку Failed Deposits (06) настроена на T+30 мин

**⚠️ КРИТИЧЕСКИ ВАЖНО:** Если GR8 Tech не может различать причины ошибок (например, все ошибки возвращают общий код), логика маршрутизации рушится. Работайте с командой платформы, чтобы обеспечить точные коды причин ДО построения этой цепочки.

---

## ШАГ 1: СОЗДАНИЕ JOURNEY

**Название кампании:** `04-PFR-JOURNEY`  
**Тип:** Triggered journey (на основе событий)  
**Триггер входа:** Event = `deposit_failed`  
**Повторный вход:** Да (игрок может ошибиться несколько раз)  
**Кулдаун повторного входа:** 5 минут (избежать спама при быстрых повторных попытках)  
**Окно journey:** 30 минут (после чего → передача в цепочку 06)

## ШАГ 2: ДОБАВЛЕНИЕ ВЕТВЛЕНИЯ ПО ПРИЧИНЕ ОШИБКИ

**Позиция:** Сразу после триггера входа  
**Тип ветвления:** Многовариантное условное разделение  
**Ветвление по:** Полю `failure_reason` из payload события

| Ветвь | Условие | Касание |
|-------|---------|---------|
| A | `failure_reason = "INSUFF"` | Касание 1A — Меньшая сумма |
| B | `failure_reason = "METHOD"` | Касание 1B — Сменить метод |
| C | `failure_reason = "DECLINE"` | Касание 1C — Сменить метод |
| D | `failure_reason = "TIMEOUT"` | Касание 1D — Автоповтор |

## ШАГ 3: ПОСТРОЕНИЕ КАЖДОГО МАРШРУТА ОШИБКИ

### 3.1 Касание 1A — Восстановление при недостатке средств

| Настройка | Значение |
|-----------|----------|
| **Campaign ID** | `04-PFR-01A-POPUP` |
| **Канал** | InApp-попап в сессии |
| **Время** | В течение 5 секунд после ошибки |

**Содержимое попапа:**
- **Banner Text (ES):** `Probá con un Monto Menor`
- **Описание баннера:** Компактный попап восстановления — иконка кошелька со стрелкой вниз, дружелюбная визуализация «попробуй меньше», три предустановленные суммы выделены, спокойные сине-зелёные тона
- **Тело:** Три кнопки с предустановленными суммами отображены крупно
- **Кнопки:**
  | Текст кнопки | Действие |
  |-------------|----------|
  | `10,500 ARS` | Депозит в один клик → `{base_url}/cashier?amount=500` |
  | `21,000 ARS` | Депозит в один клик → `{base_url}/cashier?amount=1000` |
  | `32,000 ARS` | Депозит в один клик → `{base_url}/cashier?amount=1500` |
- **Закрытие:** Текстовая ссылка "Cancelar" под кнопками

**Логика:**
- Сумма, которую пытался внести игрок, вероятно, превышала его баланс
- Предустановленные суммы — типичные начальные значения для аргентинского рынка
- Действие в один клик снижает трение — игроку не нужно повторно вводить платёжные данные

**Шаги настройки:**
1. На ветви A добавить узел попапа
2. Настроить попап с 3 CTA-кнопками (каждая — отдельный deep link на депозит с предзаполненной суммой)
3. Установить задержку срабатывания попапа ≤5 секунд
4. Отслеживать событие `deposit_success` после показа попапа
5. Если успех → Касание 2 (празднование)
6. Если повторная ошибка или бездействие в течение 30 мин → передача в цепочку 06

### 3.2 Касание 1B — Восстановление при недоступности метода

| Настройка | Значение |
|-----------|----------|
| **Campaign ID** | `04-PFR-01B-POPUP` |
| **Канал** | InApp-попап в сессии |
| **Время** | В течение 5 секунд после ошибки |

**Содержимое попапа:**
- **Banner Text (ES):** `Probá con Otro Método`
- **Описание баннера:** Попап смены метода — иконка банковской карты со стрелками обновления/замены, логотипы альтернативных платёжных систем (Mercado Pago, банковский перевод), поддерживающие зелёные акценты
- **Body (ES):** `Tu método de pago no está disponible en este momento. Elegí otro:`
- **Кнопки:** Динамические — показать 2–3 других активных метода оплаты, доступных игроку
  | Кнопка | Действие |
  |--------|----------|
  | `{alt_method_1_name}` | `{base_url}/cashier?method={alt_method_1_id}` |
  | `{alt_method_2_name}` | `{base_url}/cashier?method={alt_method_2_id}` |
  | `{alt_method_3_name}` (если есть) | `{base_url}/cashier?method={alt_method_3_id}` |
- **Закрытие:** Текстовая ссылка "Cancelar"

**Необходимые данные:**
- Вызов API платёжных методов для получения альтернатив (исключая неудавшийся метод)
- Связь ID методов с отображаемыми именами (например, "Mercado Pago", "Tarjeta de crédito", "Transferencia")

**Шаги настройки:**
1. На ветви B вызвать API платёжных методов для получения альтернатив
2. Показать попап с динамическими кнопками для каждого доступного метода
3. Каждая кнопка открывает кассу с предвыбранным методом
4. Сумма из оригинальной попытки сохраняется
5. Отслеживать `deposit_success` → Касание 2
6. Ошибка или таймаут → цепочка 06

### 3.4 Касание 1D — Восстановление при таймауте / технической ошибке

| Настройка | Значение |
|-----------|----------|
| **Campaign ID** | `04-PFR-01D-POPUP` |
| **Канал** | InApp-попап в сессии (автоповтор) |
| **Время** | В течение 5 секунд после ошибки |

**Содержимое попапа — Фаза 1 (Автоповтор):**
- **Banner Text (ES):** `Reintentando...`
- **Описание баннера:** Визуализация загрузки/повтора — круговая анимация спиннера, иконка шестерёнки, спокойная синяя тема обработки, индикатор прогресса
- **Text (ES):** `Reintentando…`
- **Поведение:** Автоматический повтор того же депозита с той же суммой и методом через 10 секунд
- **Действие игрока не требуется**

**Содержимое попапа — Фаза 2 (Если автоповтор не удался):**
- **Banner Text (ES):** `Problema Técnico`
- **Описание баннера:** Попап технической проблемы — иконка гаечного ключа/шестерёнки с треугольником предупреждения, две чёткие кнопки действий ниже, поддерживающие янтарно-синие тона
- **Body (ES):** `Podés reintentar o probar con otro método:`
- **Button 1 (ES):** `Reintentar` → повтор того же депозита
- **Button 2 (ES):** `Otro método` → показать альтернативы (аналогично 1B)
- **Закрытие:** Текстовая ссылка "Cancelar"

**Шаги настройки:**
1. На ветви D запустить автоповтор через 10 секунд
2. Показать попап со спиннером и текстом "Reintentando…"
3. Если автоповтор успешен → Касание 2 (празднование)
4. Если автоповтор неудачен → заменить спиннер на попап с ручным повтором + альтернативами
5. Отслеживать `deposit_success` → Касание 2
6. Нет успеха в течение 30 мин → цепочка 06

## ШАГ 5: ПЕРЕДАЧА В ЦЕПОЧКУ 06 (Failed Deposits)

| Настройка | Значение |
|-----------|----------|
| **Время** | T+30 минут после первоначальной ошибки |
| **Условие** | Нет успешного депозита в окне восстановления |

**Шаги настройки:**
1. Через 30 минут без события `deposit_success` → завершить эту journey
2. Пометить игрока тегом `payment_failed_unrecovered = true` и `last_failure_reason = {reason}`
3. Цепочка 06 (Failed Deposits) подхватывает с T+5 мин со своей лестницей коммуникаций

## ШАГ 6: ЧЕК-ЛИСТ ТЕСТИРОВАНИЯ

- [ ] Попап срабатывает в течение 5 секунд после события `deposit_failed` (измерить задержку)
- [ ] Ветвление корректно маршрутизирует по кодам причин ошибок
- [ ] Касание 1A: Кнопки предустановленных сумм (500, 1000, 1500) работают как депозиты в один клик
- [ ] Касание 1B: Альтернативные методы оплаты загружаются динамически и корректны
- [ ] Касание 1C: Те же альтернативы методов, что и в 1B, отображается другой текст
- [ ] Касание 1D: Автоповтор срабатывает через 10 секунд
- [ ] Касание 1D: Если автоповтор неудачен, показываются ручной повтор + альтернативы
- [ ] Касание 2: Праздничный попап появляется при успешном восстановлении
- [ ] Касание 2: 100 CC начислены на баланс игрока
- [ ] Кулдаун повторного входа (5 мин) предотвращает спам попапами при быстрых ошибках
- [ ] Передача в цепочку 06 на T+30 мин для невосстановленных ошибок
- [ ] Попапы не появляются, если игрок НЕ в сессии (edge case: игрок закрыл приложение во время повтора)
- [ ] Частотный лимит: попапы восстановления ИСКЛЮЧЕНЫ из дневных лимитов сообщений (транзакционные)

**Нагрузочный тест:**
- [ ] Сымитировать 100 одновременных ошибок — все попапы срабатывают в течение 5 секунд
- [ ] Нет дублирующихся попапов на одно событие ошибки
