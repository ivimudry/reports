# 09 — Кластер Персонализации — Руководство по внедрению

**Цепочка:** Персонализация (Anniversary, Birthday, Custom, Zodiac)  
**Расчётное количество коммуникаций:** 4 на тип × 4 типа = 16 всего  
**Необходимые данные из мастер-документа:** Атрибуты `registration_date`, `birthday`, `zodiac_sign`, `most_played_game_name`, `biggest_win_ars`, `total_rounds_played`

---

## ШАГ 0: ПРЕДСТАРТОВЫЙ ЧЕК-ЛИСТ

- [ ] Атрибут `registration_date` доступен (для расчёта годовщины)
- [ ] Атрибут `birthday` доступен (из KYC или профиля)
- [ ] Атрибут `zodiac_sign` рассчитан на основе даты рождения
- [ ] Статистика игрока доступна: общее количество раундов, любимая игра, крупнейший выигрыш (для Email годовщины)
- [ ] Бонусные шаблоны созданы: Anniversary Year 1 (100% до 5 000 + 30 FS + 500 CC), Birthday (100% до 3 000 + 25 FS + 300 CC)
- [ ] Шаблон бонуса Zodiac: 50% матч + 15 FS, вейджер 20×, срок действия 7 дней
- [ ] Lucky Wheel доступен для игр, соответствующих знаку зодиака
- [ ] CRM-менеджер может вручную запускать кампании "Personal Custom"

## ТИП C: PERSONAL CUSTOM (ПЕРСОНАЛЬНЫЕ КАСТОМНЫЕ)

**Название Journey:** Ручной запуск (без автоматизированной Journey)  
**Действие CRM-менеджера:** Создать разовую кампанию, используя структуру шаблона

### Шаблон для каждой кастомной кампании:

| Касание | Время | Канал | Содержание |
|---------|-------|-------|------------|
| C1 | Зависит от триггера | Лучший канал для игрока | Первое персонализированное сообщение |
| C2 | +24–48 ч | Канал для фоллоу-апа | Напоминание, если нет ответа |
| C3 | +72 ч | Финальный канал | Последнее напоминание перед истечением предложения |
| C4 | +96 ч | Email | Благодарность или переход |

**Обязательные поля для каждой кампании:**
- Условие триггера (какое событие или ручной отбор)
- Размер целевой аудитории
- Канал для каждого касания
- Текст для каждого касания на испанском
- Детали предложения (% матча, FS, CC, вейджер, срок действия)
- Условие выхода
- Метрика успеха
- **Ревью: Еженедельная CRM-встреча**

## ЧЕК-ЛИСТ ТЕСТИРОВАНИЯ

- [ ] Anniversary триггерится в правильную дату (годовщина `registration_date`)
- [ ] Anniversary Year 2+ корректно увеличивает предложение
- [ ] Статистика игрока в Email годовщины точна
- [ ] Birthday триггерится только для игроков с подтверждённым полем `birthday`
- [ ] Birthday мультиканальность (Email + popup + SMS) срабатывает одновременно
- [ ] Знак зодиака рассчитывается корректно на основе даты рождения
- [ ] Zodiac кампания таргетирует только игроков с соответствующим знаком
- [ ] Custom кампании могут быть запущены вручную CRM-менеджером
- [ ] Все предложения имеют корректный вейджер, срок действия и суммы
- [ ] Бонусы зачисляются корректно при депозите
