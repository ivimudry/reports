# 05 — Калибровка лестницы приветственных бонусов — Руководство по внедрению

**Цепочка:** Лестница приветственных бонусов (10 депозитов × 3 вертикали)  
**Приоритет:** Высокий (исправляет обрыв D1→D2, который убивает конверсию в STD)  
**Ориентировочные конфигурации:** 30 шаблонов бонусов (10 на вертикаль) + логика триггера лестницы  
**Предварительные условия из мастер-документа:** События `deposit_success` с `deposit_number`; Атрибуты `deposit_count`, `game_preference`, `cumulative_deposits_ars`, `most_played_game_name`; API управления бонусами

---

## ШАГ 0: ПРЕДСТАРТОВЫЙ ЧЕК-ЛИСТ

- [ ] Событие `deposit_success` содержит `deposit_number` (от 1 до 10+)
- [ ] Атрибут `game_preference` корректно установлен: `slots` / `live` / `sport` / `mixed`
- [ ] API управления бонусами может создавать бонусы депозит-матч с фриспинами/бонусными деньгами/фрибетами
- [ ] Правила вклада в отыгрыш настроены: Слоты 100%, Live 10%, Столы 5%, Спорт 100%
- [ ] Максимальная ставка при отыгрыше: 1 500 ARS за спин
- [ ] Максимальный бонусный баланс: 30 000 ARS в любой момент
- [ ] Game ID подтверждены для всех целевых игр фриспинов (см. список ниже)
- [ ] Атрибут `most_played_game_name` игрока доступен для персонализации D9 и D10
- [ ] Система Lucky Wheel готова (для D9 и D10)
- [ ] API начисления CC (Cuatro Coins) доступен
- [ ] Триггер автоматического Pre-VIP enrollment при D10 + совокупная сумма ≥150K ARS

## ШАГ 2: СОЗДАНИЕ АВТОМАТИЗИРОВАННОГО ПУТЕШЕСТВИЯ ЛЕСТНИЦЫ

**Название кампании:** `05-WBL-JOURNEY`  
**Тип:** Triggered journey (на основе событий)  
**Триггер входа:** Event = `deposit_success` WHERE `deposit_number <= 10`  
**Повторный вход:** Да (срабатывает на каждом из первых 10 депозитов)

### Логика путешествия (псевдокод):

```
ON deposit_success WHERE deposit_number <= 10:
  
  1. DETERMINE VERTICAL:
     IF game_preference = "live"  → use WBL-L-D{deposit_number}
     IF game_preference = "sport" → use WBL-SP-D{deposit_number}
     ELSE                         → use WBL-S-D{deposit_number}  (default)
  
  2. APPLY BONUS:
     Call Bonus API → grant template for this deposit number
     
  3. IF deposit_number = 1 AND amount > 15000:
     Override match to 150%
  
  4. IF deposit_number >= 5:
     Credit CC (amount per template)
  
  5. IF deposit_number >= 9:
     Grant Lucky Wheel entries
  
  6. IF deposit_number = 10 AND cumulative_deposits >= 150000:
     Trigger Pre-VIP enrollment (chain 02)
  
  7. SEND NOTIFICATION:
     Show in-app popup confirming bonus grant
     Copy: "¡Bono de depósito #{deposit_number} activado! {match}% + {fs/cash/bet} + {cc} CC"
```

**Шаги настройки:**
1. Создайте triggered-кампанию по событию `deposit_success`
2. Добавьте условное ветвление по `game_preference` для выбора вертикали
3. Внутри каждой ветки вертикали добавьте условное ветвление по `deposit_number` (1–10)
4. Для каждого номера депозита назначьте соответствующий шаблон бонуса
5. Добавьте условие переопределения D1 для депозитов >15K ARS
6. Добавьте действия начисления CC для D5–D10
7. Добавьте выдачу Lucky Wheel для D9–D10
8. Добавьте триггер Pre-VIP enrollment для D10 + совокупная сумма ≥150K
9. Добавьте всплывающее подтверждение после каждой выдачи бонуса

- [ ] Бонус D1 выдаётся корректно на 120% (стандарт) и 150% (>15K)

- [ ] Определение вертикали корректно (`game_preference` → правильная лестница)
