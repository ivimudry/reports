# 08 — No Cashier Visited — Гайд з імплементації

**Ланцюг:** No Cashier Visited (зареєстровані гравці, які ніколи не відкривали сторінку депозиту)  
**Три шляхи:** Bounce (швидкий вихід) / Long Session (переглядав, без каси) / Free Spins (грав демо)  
**Очікувані комунікації:** 4 на шлях = 12 загалом  
**Передумови з Master Doc:** Подія `session_start`, `session_end`, `cashier_opened`, `game_launched`; Атрибути `last_session_duration_seconds`, `has_played_demo`, `deposit_count`; Сегменти `SEG-BOUNCE`, `SEG-LONG-NO-CASHIER`, `SEG-FS-NO-DEP`

---

## КРОК 0: ПЕРЕДСТАРТОВИЙ ЧЕК-ЛИСТ

- [ ] Подія `session_end` містить `session_duration_seconds`
- [ ] Подія `cashier_opened` спрацьовує, коли гравець переходить на сторінку депозиту
- [ ] Атрибут `has_played_demo` відстежує демо/фріспін гру
- [ ] Сегмент `SEG-BOUNCE`: сесія <60 секунд + без взаємодії з касою
- [ ] Сегмент `SEG-LONG-NO-CASHIER`: сесія >600 секунд + без каси
- [ ] Сегмент `SEG-FS-NO-DEP`: грав демо/FS + без депозиту
- [ ] Посилання на демо-ігри: `{base_url}/game/{game_id}?demo=true`
- [ ] Історія перегляду сесії доступна для персоналізації (необов'язково, для Шляху B Комунікації 2)

---

## КРОК 1: СТВОРЕННЯ ТРЬОХ JOURNEY

| Journey Name | Тригер входу | Сегмент |
|-------------|--------------|---------|
| `08-NC-BOUNCE-JOURNEY` | `session_end` WHERE `session_duration < 60` AND `deposit_count = 0` AND без події `cashier_opened` | `SEG-BOUNCE` |
| `08-NC-LONG-JOURNEY` | `session_end` WHERE `session_duration >= 600` AND `deposit_count = 0` AND без події `cashier_opened` | `SEG-LONG-NO-CASHIER` |
| `08-NC-FS-JOURNEY` | `free_spins_used` OR `game_launched` WHERE `is_demo = true` AND `deposit_count = 0` | `SEG-FS-NO-DEP` |

**Глобальний вихід для всіх:** `deposit_success` → вихід, перехід до Day 1 Retention Layer.

## КРОК 5: ЧЕК-ЛИСТ ТЕСТУВАННЯ

- [ ] Шлях A: Виявлення баунсу коректне (сесія <60с + без каси)
- [ ] Шлях B: Виявлення довгої сесії коректне (сесія >600с + без каси)
- [ ] Шлях C: Виявлення демо/FS гри коректне
- [ ] Усі 3 шляхи завершуються негайно при `deposit_success`
- [ ] Шлях A Комунікація 2: Гайд швидкого старту відображається з іконками платіжних методів
- [ ] Шлях B: Персоналізація перегляду працює (якщо доступна з даних сесії)
- [ ] Шлях C: Персоналізація гри (`{game_they_played}`) розв'язується коректно
- [ ] Шлях C Комунікація 4: 10 безвнескових FS нараховуються при кліку на посилання
- [ ] Суми підсолоджувачів: Bounce = 200 CC, Long Session = 300 CC, FS = 10 додаткових FS
- [ ] Обмеження частоти: макс. 2/день (сегмент pre-FTD)
- [ ] Без перетину між шляхами (гравець потрапляє тільки в ОДИН шлях на основі поведінки)
- [ ] UTM-параметри на всіх посиланнях
