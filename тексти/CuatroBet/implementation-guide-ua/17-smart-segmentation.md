## C. ВІСЬ 2: VALUE TIER — ПОБУДОВА ПРАВИЛ

**Ця вісь визначає РОЗМІР ПРОПОЗИЦІЇ (множник для базових пропозицій).**

### Створити сегменти:

| Назва сегменту | Визначення | Множник пропозиції |
|-------------|-----------|-----------------|
| `VT_MICRO` | `deposits_lifetime_ars < 5000` | 0.5× |
| `VT_LOW` | `deposits_lifetime_ars BETWEEN 5000 AND 25000` | 1× (базовий) |
| `VT_MID` | `deposits_lifetime_ars BETWEEN 25001 AND 75000` | 1.5× |
| `VT_HIGH` | `deposits_lifetime_ars BETWEEN 75001 AND 200000` | 2× |
| `VT_PRE_VIP_PLUS` | `deposits_lifetime_ars > 200000` | 3× (+ персональний контакт) |

### Як застосовувати:

Кожен journey, що відправляє бонус/пропозицію, зчитує value tier гравця та застосовує множник:

```
final_offer_value = base_offer × value_tier_multiplier
```

Приклад: Базова пропозиція Chain 12 = 500 ARS без депозиту
- Гравець Micro: 250 ARS
- Гравець Low: 500 ARS
- Гравець Mid: 750 ARS
- Гравець High: 1 000 ARS
- Pre-VIP+: 1 500 ARS + персональний дзвінок

**Важливо:** Вимоги вейджеру НЕ масштабуються. Масштабується тільки значення пропозиції.

## D. ВІСЬ 3: GAME PREFERENCE — ПОБУДОВА ПРАВИЛ

**Ця вісь визначає КОНТЕНТ (які ігри, FS, банери показувати).**

### Створити сегменти:

| Назва сегменту | Визначення | Прив'язка до контенту |
|-------------|-----------|----------------|
| `GP_SLOTS` | `slots_wagered / total_wagered > 0.7` | FS на топ-слоті, банери слотів, турніри слотів |
| `GP_AVIATOR` | `aviator_wagered / total_wagered > 0.3` | Free flights, контент crash-ігор |
| `GP_LIVE` | `live_wagered / total_wagered > 0.3` | Пропозиції настільних ігор, контент live dealer |
| `GP_MIXED` | Жоден з вищезгаданих не >0.7 або >0.3 | Різноманітний контент, "спробуй щось нове" |

### Бібліотека контенту (побудувати для кожної переваги):

| Перевага | FS гра | Тема банеру | Тип турніру | Рекомендація |
|-----------|---------|-------------|----------------|----------------|
| Slots | Jackpot Joker, Gates of Olympus | Яскрава, зображення джекпотів | Турніри слотів | Топ-10 за engagement |
| Aviator | Free Flights | Небо, літак, множник | Лідерборд crash-ігор | Подібні crash-ігри |
| Live | — (використовувати бонус cash) | Елегантна, зображення дилера | Турніри live-столів | Преміум столи |
| Mixed | Випадкова з топ-10 | Різноманітна | Кросігрові челенджі | Мало освоєні категорії |

## E. ВІСЬ 4: CHURN RISK — ПОБУДОВА ПРАВИЛ

**Ця вісь ПРИСКОРЮЄ час — відправляє комунікації раніше для гравців під ризиком.**

### Створити сегменти:

| Назва сегменту | Діапазон Score | Дія |
|-------------|-----------|--------|
| `CR_HEALTHY` | churn_score < 0.30 | Нормальний час |
| `CR_WATCH` | churn_score 0.30–0.60 | Відправити на 1 день раніше за замовчуванням |
| `CR_AT_RISK` | churn_score 0.60–0.85 | Відправити на 2 дні раніше; підвищити пропозицію на 1 рівень |
| `CR_CRITICAL` | churn_score > 0.85 | Негайне втручання; сповістити VIP-команду для High+ value |

### Інтеграція з Journey:

```
IF player.churn_risk = "CR_AT_RISK" AND player.lifecycle = "LS_REGULAR":
  → Увійти в Chain 12 (Reactivation) у бакет 7 днів РАНІШЕ (день 3-4)
  → Tag: "churn_predicted_early_entry"

IF player.churn_risk = "CR_CRITICAL" AND player.value_tier IN ("VT_HIGH", "VT_PRE_VIP_PLUS"):
  → Негайне внутрішнє сповіщення VIP-команді
  → Пропустити автоматизований ланцюжок, персональний контакт протягом 24 год.
  → Tag: "churn_critical_vip_intervention"
```

### Налаштування моделі Predictive Churn:

| Крок | Дія |
|------|--------|
| 1 | Зв'язатися з GR8 Tech для активації модуля Predictive Churn |
| 2 | Надати 30+ днів історичних даних (логіни, депозити, геймплей, engagement комунікацій) |
| 3 | GR8 Tech навчає модель на специфічних даних CuatroBet |
| 4 | Модель видає щоденний `churn_score` (0.0–1.0) на гравця |
| 5 | Валідувати: перевірити чи score >0.6 коректно передбачає відтік протягом наступних 14 днів |
| 6 | Якщо точність <70%, перенавчити з коригованими features |
| 7 | Підключити churn_score до правил сегментів вище |

## F. ВІСЬ 5: KYC STATUS — ПОБУДОВА ПРАВИЛ

**Ця вісь БЛОКУЄ доступність каналів та певних дій.**

### Створити сегменти:

| Назва сегменту | Визначення | Доступ до каналів |
|-------------|-----------|---------------|
| `KYC_UNVERIFIED` | Жодних кроків верифікації не завершено | Тільки Email + NC Feed |
| `KYC_PARTIAL` | Email верифіковано + телефон верифіковано | Email + SMS + NC Feed + Push |
| `KYC_FULL` | Документи верифіковано (ID, підтвердження адреси) | Усі канали + виведення коштів |

### Правила блокування каналів:

| Канал | Вимагає |
|---------|----------|
| Email | `KYC_UNVERIFIED` або вище (завжди доступний) |
| NC Feed | `KYC_UNVERIFIED` або вище |
| NC Pop-Up | `KYC_UNVERIFIED` або вище (тільки in-app) |
| SMS | `KYC_PARTIAL` або вище (потрібен номер телефону) |
| App Push | `KYC_UNVERIFIED` або вище (device token, не залежить від KYC) |
| WhatsApp | `KYC_PARTIAL` або вище |

### Інтеграція KYC → Chain 03:

Якщо `KYC_UNVERIFIED` або `KYC_PARTIAL`:
- Гравець автоматично входить у Chain 03 (KYC Completion Chain)
- Тригери Chain 03 ескалуються на основі порогів депозиту (15K soft, 22.5K hard)

## G. КОМБІНОВАНИЙ ПРИКЛАД

**Профіль гравця:**
```
Name: Juan
Lifecycle: LS_REGULAR (активний, 5 депозитів)
Value Tier: VT_MID (42 000 ARS за весь час)
Game Pref: GP_SLOTS (85% ставок на слотах)
Churn Risk: CR_WATCH (score 0.45)
KYC Status: KYC_FULL
```

**Що відбувається:**
1. **Вісь 1 (Lifecycle):** Juan у Regular lifecycle → має право на Chains 02, 07, 08, 09, 10, 11
2. **Вісь 2 (Value):** Рівень Mid → усі пропозиції × множник 1.5
3. **Вісь 3 (Game Pref):** Slots → FS на Jackpot Joker, банери слотів, контент турнірів слотів
4. **Вісь 4 (Churn Risk):** Watch → комунікації відправляються на 1 день раніше за стандартний розклад
5. **Вісь 5 (KYC):** Full → усі канали доступні (SMS, email, push, popup тощо)

**Результат:** Juan отримує сповіщення квестів Chain 10 зі слот-специфічними челенджами, пропозиція 1.5× базової кількості FS, відправлення на 1 день раніше, через бажаний канал (визначається з історії engagement з Chain 13).
