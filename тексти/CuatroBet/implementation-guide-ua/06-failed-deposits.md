# 06 — Failed Deposits: Communication Ladders — Гайд з імплементації

**Ланцюг:** Failed Deposits (подальша комунікація після відновлення)  
**Початкова точка:** T+5 хв після невдалого депозиту (ланцюг 04 обробляє T+0 до T+5хв)  
**Обсяг:** 5 причин відмови × 4 комунікації = 20 загальних комунікацій  
**Передумови з Master Doc:** Подія `deposit_failed`, `cashier_opened`, `cashier_closed`, `deposit_success`; Атрибути `deposit_count`, `welcome_bonus_claimed`; Сегменти `SEG-CASHIER-ABANDONED`

- [ ] Подія `cashier_closed` спрацьовує, коли гравець залишає сторінку депозиту без депозиту

## КРОК 1: СТВОРЕННЯ 5 ОКРЕМИХ JOURNEY

Створіть один journey на кожну причину відмови. Це зберігає чистоту логіки та дозволяє незалежну оптимізацію.

| Journey Name | Тригер входу | Час початку |
|-------------|--------------|-------------|
| `06-FAIL-R1-JOURNEY` | `cashier_closed` + жодної спроби депозиту + 30 хв бездіяльності | T+30 хв |
| `06-FAIL-R2-JOURNEY` | `deposit_failed` + `failure_reason = TIMEOUT/METHOD` + attempt_count = 1 + не відновлено в ланцюгу 04 | T+5 хв |
| `06-FAIL-R3-JOURNEY` | `deposit_failed` + `attempt_count_session >= 2` + технічна причина + не відновлено | T+5 хв |
| `06-FAIL-R4-JOURNEY` | `deposit_failed` + `failure_reason = DECLINE/INSUFF` + не відновлено | T+5 хв |
| `06-FAIL-R5-JOURNEY` | `deposit_failed` + `failure_reason = FRAUD` | T+5 хв |

**Глобальний вихід для всіх journey:** подія `deposit_success` → негайний вихід з journey.

## КРОК 7: ЧЕК-ЛИСТ ТЕСТУВАННЯ

- [ ] Кожна причина коректно маршрутизується до свого journey
- [ ] Глобальний вихід спрацьовує при `deposit_success` (зупиняє всі комунікації негайно)
- [ ] Причина 1: Закриття каси + 30 хв бездіяльності запускає journey
- [ ] Причина 2: Одна технічна відмова + без відновлення → journey стартує на T+5хв
- [ ] Причина 3: Виявлено 2+ відмов → тікет підтримки автоматично створено
- [ ] Причина 3: Бажаний канал зв'язку гравця записано та використано
- [ ] Причина 4: Альтернативні платіжні методи показуються динамічно
- [ ] Причина 5: Акаунт позначено → черга перевірки шахрайства
- [ ] Причина 5: Комунікації 3–4 спрацьовують тільки для хибних спрацювань
- [ ] Кредити-вибачення (200 ARS / 500 ARS) нараховуються коректно
- [ ] Кількість символів SMS ≤160 для всіх повідомлень
- [ ] UTM-параметри на всіх посиланнях
- [ ] Обмеження частоти дотримуються (макс. 2/день для pre-FTD)
- [ ] Немає перетину з ланцюгом 04 (PFR обробляє тільки T+0 до T+5хв)
