# 02 — Життєвий цикл Pre-VIP — Інструкція з впровадження

**Ланцюг:** Життєвий цикл Pre-VIP  
**Пріоритет:** Високий  
**Орієнтовна кількість комунікацій:** 6 кроків + точки рішень протягом 30 днів  
**Передумови з основного документа:** Атрибути `cumulative_deposits_ars`, `days_since_last_activity`, `game_preference`, `most_played_game_name`; Сегменти `SEG-PREVIP`, `SEG-VIP`

---

## КРОК 0: ПЕРЕДПОЛЬОТНИЙ ЧЕК-ЛИСТ

- [ ] Атрибут `cumulative_deposits_ars` оновлюється в реальному часі при кожному депозиті
- [ ] Атрибут `days_since_last_activity` розраховується щоденно
- [ ] Система призначення CRM-менеджера готова (іменна особа, реальний email, reply-to)
- [ ] Бонусний шаблон 15% тижневого кешбеку створено (без відіграшу, виплата по понеділках, ліміт 30,000 ARS)
- [ ] Бонусний шаблон для Кроку 3: 60% матч до 10,000 ARS + 30 FS, 20× відіграш, 72г термін дії
- [ ] Бонусний шаблон для Кроку 5: 100% матч до 20,000 ARS + 50 FS, 15× відіграш, 72г термін дії
- [ ] Модуль Predictive Churn активовано (для моніторингу після Дня 21+)
- [ ] Дашборд Pre-VIP існує у CRM для моніторингу командою
- [ ] Дані про улюблений слот гравця (`most_played_game_name`) доступні

---

## КРОК 1: СТВОРЕННЯ ПОДОРОЖІ

**Назва кампанії:** `02-PVIP-JOURNEY`  
**Тип:** Тригерна подорож (на основі атрибутів)  
**Тригер входу:** `cumulative_deposits_ars >= 150000` AND `days_since_last_activity <= 14`  
**Повторний вхід:** Немає  
**Вікно подорожі:** 30 днів (можна продовжити до 60)  
**Умови виходу:**
- Гравець досягає `cumulative_deposits_ars >= 300000` (VIP I) → вихід до онбордингу VIP I
- Минає 60 днів без VIP I → перехід до звичайного життєвого циклу

## КРОК 2: ПОБУДОВА ПОСЛІДОВНОСТІ КРОКІВ

### 2.1 Крок 1 — Активація Pre-VIP (Персональний Email)

| Параметр | Значення |
|---------|-------|
| **ID кампанії** | `02-PVIP-T1-EMAIL` |
| **Канал** | Email |
| **Час** | День 1 (при вході в подорож) |
| **Від** | Іменний CRM-менеджер (напр., "Matias de CuatroBet") — НЕ загальний |
| **Reply-to** | Реальний email CRM-менеджера (або моніторена скринька) |

**Системні дії (перед відправкою email):**
1. Призначити іменного CRM-менеджера до запису гравця
2. Позначити гравця як Pre-VIP у сегментації
3. Додати гравця до дашборду моніторингу Pre-VIP

**Контент Email:**
- **Тема (ES):** `👑 Acabas de desbloquear estatus Pre-VIP en CuatroBet`
- **Прехедер (ES):** `Soy {crm_owner_name}, tu manager personal`
- **Текст банера (ES):** `Desbloqueaste Pre-VIP`
- **Опис банера:** Преміальний банер у золотих тонах — бейдж Pre-VIP з іконкою корони, темний розкішний фон, золоті ефекти частинок, місце для імені гравця
- **Опис тіла:** Персональне привітання від іменного CRM-менеджера. Вітання з Pre-VIP статусом. Перелік переваг: 15% тижневий кешбек без відіграшу, персоналізовані рекомендації ігор, швидші виведення, ексклюзивні пропозиції. Згадка про близькість до VIP I. Привітальний подарунок: 500 CC + 20 FS на улюбленому слоті гравця. Розмовний тон, без жорсткого CTA — підписано CRM-менеджером з ім'ям, посадою, фото.
- **CTA:** Немає (розмовний тон, без жорсткого CTA)
- **Підпис:** Ім'я CRM-менеджера, посада, фото за наявності

**Конфігурація бонусу:**
- 500 CC (Cuatro Coins) — зарахувати негайно
- 20 FS на улюбленому слоті гравця (`most_played_game_name`)

**Дія в додатку:**
- Показати банер розблокування Pre-VIP при наступному вході гравця

**Кроки конфігурації:**
1. Додайте вузол email на початку подорожі
2. Використовуйте токени персоналізації: `{first_name}`, `{crm_owner_name}`, `{player_favorite_slot}`
3. Встановіть відправника як CRM-менеджера (не загальний "CuatroBet")
4. Додайте паралельну системну дію: Зарахувати 500 CC + 20 FS
5. Додайте паралельну дію: Запланувати in-app банер для наступного входу

- **Прехедер (ES):** `15% semanal, sin wagering, cada lunes`
- **Текст банера (ES):** `15% Cashback Semanal`
- **Опис банера:** Візуал кешбеку — великий «15%» золотом на темному фоні, монети, що повертаються до іконки гравця, іконка тижневого календаря, бейдж «Sin Wagering»
- **Опис тіла:** Пояснення механіки кешбеку: розраховується на тижневі чисті збитки, зараховується автоматично кожного понеділка, нуль відіграшу, ліміт 30,000 ARS на тиждень. Заспокійливий тон.

**Системна дія:**
- Увімкнути правило 15% тижневого кешбеку для цього гравця (термін 30 днів, без відіграшу, виплата по понеділках, ліміт 30,000 ARS)

**In-App банер:**
- Текст: "15% Cashback Activo — Sin wagering"
- Показувати 7 днів при вході

**Кроки конфігурації:**
1. Додайте затримку 2 дні (День 1 → День 3)
2. Додайте вузол email
3. Додайте системну дію: активувати правило кешбеку
4. Додайте in-app банер (відображення 7 днів)

4. Шлях «Ні» → Продовження

## КРОК 3: ЧЕК-ЛИСТ ТЕСТУВАННЯ

- [ ] Подорож спрацьовує коли сукупні депозити перетинають 150,000 ARS
- [ ] Подорож НЕ спрацьовує для гравців неактивних >14 днів
- [ ] CRM-менеджер призначається коректно при вході
- [ ] Email Кроку 1 надходить від CRM-менеджера (не загального відправника)
- [ ] 500 CC + 20 FS зараховані при Кроці 1
- [ ] Кешбек активується на День 3 (перевірити, що понеділкова виплата працює)
- [ ] Рекомендації ігор у Кроці 3 відповідають історії гравця
- [ ] Push Кроку 4 показує правильну залишкову суму
- [ ] Рішення Дня 14 коректно визначає кваліфікацію VIP I
- [ ] Бонус Кроку 5 (100% матч) доступний тільки не-VIP гравцям
- [ ] Моніторинг відтоку Дня 21 активується для гравців з депозитами
- [ ] Вихід до онбордингу VIP I працює у будь-який момент подорожі
- [ ] Розширене 30-денне вікно має зменшену каденцію
- [ ] Усі email від CRM-менеджера мають правильний reply-to
