# 07 — No Bonus Cluster — Гайд з імплементації

**Ланцюг:** No Bonus Cluster (гравці, які не отримали вітальний бонус)  
**Дві гілки:** Missed (не побачили) vs None Interested (активно відхилили)  
**Очікувані комунікації:** 4 на гілку = 8 загалом  
**Передумови з Master Doc:** Атрибути `welcome_bonus_claimed`, `welcome_bonus_status`; Сегменти `SEG-BONUS-MISSED`, `SEG-BONUS-DECLINED`

---

## КРОК 0: ПЕРЕДСТАРТОВИЙ ЧЕК-ЛИСТ

- [ ] Атрибут `welcome_bonus_claimed` відстежує, чи отримав гравець вітальний бонус
- [ ] Атрибут `welcome_bonus_status` розрізняє: `available` / `claimed` / `declined` / `expired`
- [ ] Логіка виявлення «бонус оминутий» існує (гравець прокликав потік бонусу без отримання)
- [ ] Посилання на демо/безкоштовну гру працюють: `{base_url}/game/{game_id}?demo=true`
- [ ] Список Топ-5 ігор доступний для персоналізації емейлу
- [ ] Бонусні шаблони 200 CC / 300 CC створені (для підсолоджувачів у Комунікаціях 4)

---

## КРОК 1: СТВОРЕННЯ ДВОХ JOURNEY

| Journey Name | Тригер входу | Сегмент |
|-------------|--------------|---------|
| `07-NB-MISS-JOURNEY` | `deposit_count = 0` AND `welcome_bonus_claimed = false` AND `hours_since_registration >= 1` AND жодної взаємодії з бонусом не виявлено | `SEG-BONUS-MISSED` |
| `07-NB-NONE-JOURNEY` | `welcome_bonus_status = "declined"` OR потік бонусу оминуто | `SEG-BONUS-DECLINED` |

**Глобальний вихід для обох:** `deposit_success` → вихід, перехід до Day 1 Retention Layer.

## КРОК 4: ЧЕК-ЛИСТ ТЕСТУВАННЯ

- [ ] Гілка A правильно ідентифікує гравців, які ПРОПУСТИЛИ бонус (без взаємодії, не відхилено)
- [ ] Гілка B правильно ідентифікує гравців, які ВІДХИЛИЛИ бонус (явна дія або обхід)
- [ ] Гілка A: Усі 4 комунікації прямо згадують вітальний бонус
- [ ] Гілка B: Комунікації 1 та 3 НЕ мають згадки бонусу (тільки відкриття продукту)
- [ ] Гілка B: Комунікації 2 та 4 мають лише М'ЯКУ згадку бонусу у футері
- [ ] Демо-посилання працюють коректно: `?demo=true` відкриває гру у безкоштовному режимі
- [ ] Глобальний вихід спрацьовує миттєво при `deposit_success`
- [ ] Підсолоджувач 200 CC у Гілці A Комунікації 4 нараховується коректно
- [ ] Обмеження частоти: макс. 2 повідомлення/день (сегмент pre-FTD)
- [ ] UTM-параметри на всіх посиланнях
