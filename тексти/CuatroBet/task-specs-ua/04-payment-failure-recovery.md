# 04 — Відновлення Невдалих Платежів

**Пріоритет:** 🟡 Високий  
**Мета:** Відновити 15–25% невдалих спроб депозиту в реальному часі (перші 30 секунд у сесії)  
**Тригер входу:** Подія невдалого депозиту в реальному часі від GR8 Tech (спрацьовує протягом 5 секунд)  
**Доповнює:** Кластер Failed (Частина II, секція 6), який обробляє від T+5хв і далі

---

## Класифікація Причин Невдачі

Платформа має надавати код причини невдачі. Чотири маршрути:

| Код | Причина | Опис |
|-----|---------|------|
| `INSUFF` | Недостатньо коштів | Гравець намагався внести більше, ніж доступний баланс |
| `METHOD` | Метод недоступний | Платіжний метод тимчасово не працює або не підтримується |
| `DECLINE` | Відхилено еквайєром | Банк або платіжний провайдер відхилив транзакцію |
| `TIMEOUT` | Таймаут / технічна | Таймаут шлюзу, мережева помилка, невідомо |

---

### Тач 1A — Відновлення при Недостатньому Балансі
| Поле | Значення |
|------|----------|
| **ID** | PFR-01A |
| **Таймінг** | T+0 (протягом 5 секунд після невдачі) |
| **Канал** | Попап у сесії |
| **Тригер** | Депозит невдалий, причина = Недостатньо коштів |
| **Сегмент** | Гравці з невдалим депозитом (недостатньо коштів) |
| **Копі (ES)** | "Probá con un monto menor" |
| **Дія** | Показати кнопки з пресетами: **500 / 1,000 / 1,500 ARS** |
| **CTA** | Депозит в один клік на меншу суму |
| **Вихід** | Депозит успішний → Тач 2 (святкування); Знову невдача → передача до кластеру Failed через T+30хв |
| **Примітки** | Гравець ймовірно хотів внести 5,000, а має лише 1,500. Менший пресет вирішує це. |

---

### Тач 1B — Відновлення при Недоступному Методі
| Поле | Значення |
|------|----------|
| **ID** | PFR-01B |
| **Таймінг** | T+0 (протягом 5 секунд) |
| **Канал** | Попап у сесії |
| **Тригер** | Депозит невдалий, причина = Метод недоступний (Mercado Pago / Naranja / банк не працює) |
| **Копі (ES)** | "Probá con otro método" |
| **Дія** | Кнопки в один клік для інших 2-х активних платіжних методів, попередньо вибрані |
| **CTA** | Вибрати альтернативний метод → повторити |
| **Вихід** | Депозит успішний → Тач 2; Невдача → кластер Failed |
| **Примітки** | Безфрикційна заміна. Гравцю не потрібно шукати методи в меню. |

| Поле | Значення |
|------|----------|
| **ID** | PFR-01C |
| **Таймінг** | T+0 (протягом 5 секунд) |
| **Канал** | Попап у сесії |
| **Тригер** | Депозит невдалий, причина = Відхилено еквайєром |
| **Копі (ES)** | "Tu banco rechazó el pago. Probá con otro método" |

| **Вихід** | Депозит успішний → Тач 2; Невдача → кластер Failed |
| **Примітки** | Проблема в банку. Допоможе лише зміна методу. |

### Тач 1D — Відновлення при Таймауті / Технічній Помилці
| Поле | Значення |
|------|----------|
| **ID** | PFR-01D |
| **Таймінг** | T+0 (протягом 5 секунд) |

| **Тригер** | Депозит невдалий, причина = Таймаут / технічна помилка |
| **Дія** | Авто-повтор через 10 секунд зі спінером "Reintentando…" |
| **Копі (ES)** | "Reintentando…" (спінер) |
| **CTA** | — (автоматично) |
| **Вихід** | Авто-повтор успішний → Тач 2; Невдача → показати кнопку ручного повтору + альтернативні методи |
| **Примітки** | Таймаути шлюзу часто вирішуються при повторі. Жодного повторного введення від гравця не потрібно. |

### Тач 2 — Святкування Відновлення (усі причини)
| Поле | Значення |
|------|----------|
| **ID** | PFR-02 |
| **Таймінг** | Одразу при успішному відновленні |
| **Канал** | Попап у сесії |
| **Тригер** | Депозит успішний після потоку відновлення |
| **Сегмент** | Гравці, що відновились після невдалого депозиту |
| **Дія** | Святкування + подарунок |
| **Оффер** | Подарунок 100 CC (Cuatro Coins) |
| **Копі (ES)** | Попап зі святкуванням |
| **Вихід** | ВИХІД → Day 1 Retention Layer (якщо FTD) або поточний lifecycle |

### Передача — Не Відновлено
| Поле | Значення |
|------|----------|
| **Таймінг** | T+30 хв (немає відновлення в сесії) |
| **Дія** | Передати до комунікаційної драбини кластеру Failed (секція 06) |
| **Примітки** | Кластер Failed підхоплює від T+5хв з 5 причинами × 4 комунікації кожна |

## Модель Витрат
| Параметр | Значення |
|----------|----------|
| Вартість | Найдешевша ініціатива у плейбуці |
| Важіль | Найбільше плече на кожне витрачене песо |
| 100 CC подарунок за відновлення | Незначно |

## Цільові KPI
| Метрика | Базова | Ціль |
|---------|--------|------|
| Успішність платежів (ефективна, вкл. відновлені) | 55% | 65–70% |
| FTD конверсія (нижче по воронці) | — | +2–4 пп |
| Бонусна вартість | — | Без прямого впливу (проблема платежів) |

## Залежності Реалізації
1. Подія невдачі в реальному часі від GR8 Tech (затримка менше 5 секунд)
2. Класифікація причин невдачі має бути точною (якщо платформа не може відрізнити недостатньо коштів від відхилення еквайєром, маршрутизація руйнується)
3. Працювати з командою платежів паралельно для підняття сирого показника 55% — CRM-відновлення — це стеля, реальне рішення upstream
