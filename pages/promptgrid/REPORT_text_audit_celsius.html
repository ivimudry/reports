2. Audit Rules

  • If an email has a bonus, the promo code MUST be in the body using <strong class="promocode">CODE</strong>
  • If an email has a promo code, it MUST also be in the header as data-promocode="CODE"
  • promocode_button is ONLY expected in DEP Retention — absence elsewhere is NOT an error
  • Social proof emails (top winners lists) don't require bonuses or promo codes
  • Some emails intentionally have "no code needed" — these are flagged as INFO, not errors

2. Правила аудиту

  • Якщо в листі є бонус, промокод ПОВИНЕН бути в тілі у форматі <strong class="promocode">КОД</strong>
  • Якщо в листі є промокод, він ПОВИНЕН бути в хедері як data-promocode="КОД"
  • promocode_button очікується ТІЛЬКИ в DEP Retention — відсутність в інших кампаніях НЕ є помилкою
  • Листи social proof (списки переможців) не потребують бонусів чи промокодів
  • Деякі листи навмисно мають "no code needed" — вони позначені як INFO, не помилки

2. Правила аудита

  • Если в письме есть бонус, промокод ДОЛЖЕН быть в теле в формате <strong class="promocode">КОД</strong>
  • Если в письме есть промокод, он ДОЛЖЕН быть в заголовке как data-promocode="КОД"
  • promocode_button ожидается ТОЛЬКО в DEP Retention — отсутствие в других кампаниях НЕ является ошибкой
  • Письма social proof (списки победителей) не требуют бонусов или промокодов
  • Некоторые письма намеренно имеют "no code needed" — они отмечены как INFO, не ошибки
