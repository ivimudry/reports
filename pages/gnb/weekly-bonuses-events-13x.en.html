1. Event logic

All 7 weekly campaigns use the same event model:

  1. A dedicated event exists for each 13.x campaign.
  2. The event is broadcast to all existing users at a fixed day and time.
  3. The event repeats every week on the same schedule.
  4. The campaign listens to that event as its trigger.

Test scope: this document only describes the event-based version of Weekly Bonuses. It does not change existing segments, campaigns, attributes, or current implementations anywhere else.
