active_player_period — атрибут Customer.io

Что это: период суток, когда игрок чаще всего заходит в казино (логин на сайт казино). Используется маркетингом для отправки рассылок в активное время игрока.

Источник данных: логи входов игрока в казино (session.start_time или эквивалент — событие успешного логина). Длительность сессии, ставки, депозиты — не учитываем. Только сам факт входа.

Алгоритм

  1. Берём последние 30 входов в казино (поле session.start_time, UTC).
  2. Если входов < 5 или у игрока нет таймзоны → unknown, выходим.
  3. Каждый start_time переводим в локальное время игрока, берём только час (0–23).
  4. Каждый вход даёт очки бакетам:
    • +1.0 в свой бакет;
    • +0.5 в предыдущий бакет;
    • +0.5 в следующий бакет.
    Бакеты идут по кругу: после night снова late_night.
  5. Победитель — бакет с наибольшей суммой очков. Это и есть active_player_period.
  6. Если несколько бакетов набрали поровну — см. раздел «Tie-break» ниже.
  7. Записать в Customer.io через API.

Tie-break (что делать при равенстве очков)

  1. Сравнить «сырые» входы. Побеждает бакет, в который попало больше реальных логинов (без бонусов от соседей).
  2. Если снова равенство — фиксированный приоритет периодов:
    1. evening
    2. night
    3. afternoon
    4. morning
    5. early_morning
    6. late_night
    Логика: вечер/ночь — пиковое время трафика в казино, безопасный дефолт для рассылки.
  3. Если и тут равенство (крайне маловероятно) → unknown.
Зачем «+0.5 соседям»: вход в 15:55 формально попадает в afternoon, но фактически он почти в evening. Соседи получают половину очка, чтобы границы бакетов не били результат. Если у игрока есть один чёткий период — он всё равно победит. Если активность размазана между двумя соседями — победит тот, к которому игрок ближе.
Важно: бакеты замкнуты в круг. Сосед late_night слева — это night, сосед night справа — late_night. Иначе игрок, заходящий в 23:00 и 01:00, получит ерунду.

Маппинг час → бакет

bucket_index = floor(hour / 4)
0 → late_night
1 → early_morning
2 → morning
3 → afternoon
4 → evening
5 → night

Python

BUCKETS = ['late_night', 'early_morning', 'morning',
           'afternoon', 'evening', 'night']
PRIORITY = ['evening', 'night', 'afternoon',
            'morning', 'early_morning', 'late_night']

def active_period(hours):
    """hours — список локальных часов входов в казино (0..23)."""
    if len(hours) < 5:
        return 'unknown'

    score = [0.0] * 6
    raw = [0] * 6
    for h in hours:
        i = h // 4
        raw[i] += 1
        score[i] += 1.0
        score[(i - 1) % 6] += 0.5
        score[(i + 1) % 6] += 0.5

    max_score = max(score)
    candidates = [i for i, s in enumerate(score) if s == max_score]
    if len(candidates) == 1:
        return BUCKETS[candidates[0]]

    # Tie-break 1: больше «сырых» входов
    max_raw = max(raw[i] for i in candidates)
    candidates = [i for i in candidates if raw[i] == max_raw]
    if len(candidates) == 1:
        return BUCKETS[candidates[0]]

    # Tie-break 2: фиксированный приоритет
    for name in PRIORITY:
        if BUCKETS.index(name) in candidates:
            return name

    # Tie-break 3: совсем редкий случай
    return 'unknown'

# пример: входы преимущественно вечером
print(active_period([16, 17, 18, 19, 20, 15, 19, 21]))  # evening

SQL (PostgreSQL)

WITH buckets AS (
  SELECT
    user_id,
    FLOOR(EXTRACT(HOUR FROM start_time AT TIME ZONE user_tz) / 4)::INT AS b
  FROM sessions
  WHERE start_time > NOW() - INTERVAL '30 days'
),
counts AS (
  SELECT user_id, b, COUNT(*) AS c
  FROM buckets
  GROUP BY user_id, b
),
-- разносим очки: +1 в свой бакет, +0.5 в соседей (по кругу)
scored AS (
  SELECT user_id, b AS bucket, c::FLOAT AS score FROM counts
  UNION ALL
  SELECT user_id, (b + 1) % 6, c * 0.5 FROM counts
  UNION ALL
  SELECT user_id, (b + 5) % 6, c * 0.5 FROM counts  -- (b - 1 + 6) mod 6
),
raw_counts AS (
  SELECT user_id, b AS bucket, c AS raw FROM counts
),
totals AS (
  SELECT
    s.user_id,
    s.bucket,
    SUM(s.score) AS total,
    COALESCE(MAX(r.raw), 0) AS raw,
    ROW_NUMBER() OVER (
      PARTITION BY s.user_id
      ORDER BY
        SUM(s.score) DESC,                    -- основной балл
        COALESCE(MAX(r.raw), 0) DESC,         -- tie-break 1: сырые входы
        CASE s.bucket                         -- tie-break 2: приоритет
          WHEN 4 THEN 1  -- evening
          WHEN 5 THEN 2  -- night
          WHEN 3 THEN 3  -- afternoon
          WHEN 2 THEN 4  -- morning
          WHEN 1 THEN 5  -- early_morning
          WHEN 0 THEN 6  -- late_night
        END
    ) AS rn
  FROM scored s
  LEFT JOIN raw_counts r ON r.user_id = s.user_id AND r.bucket = s.bucket
  GROUP BY s.user_id, s.bucket
),
qualified AS (
  SELECT user_id FROM buckets GROUP BY user_id HAVING COUNT(*) >= 5
)
SELECT
  q.user_id,
  CASE t.bucket
    WHEN 0 THEN 'late_night'
    WHEN 1 THEN 'early_morning'
    WHEN 2 THEN 'morning'
    WHEN 3 THEN 'afternoon'
    WHEN 4 THEN 'evening'
    WHEN 5 THEN 'night'
  END AS active_player_period
FROM qualified q
JOIN totals t ON t.user_id = q.user_id AND t.rn = 1;

Расписание

Батч-джоб раз в сутки в 03:00 UTC. Считает по всем игрокам, у которых были входы в казино за последние 30 дней, пушит результат в Customer.io через API.
