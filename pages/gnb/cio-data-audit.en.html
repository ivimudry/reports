CIO Data Audit (EN)

Chill4Reel — CIO Data Audit (EN)

How to read the tables: the Status column is a traffic light:

  • ✅ Ready — exists in CIO today, usable as-is.
  • ⚠️ Replacement — playbook asked for one thing; CIO/backend already has something else that fully covers the case. Mapping + justification recorded.
  • ❌ Integrate — does not exist in either CIO or backend; must be created (backend / CIO segment / CIO promo code).

0. Executive summary

Category ✅ Ready ⚠️ Replacement ❌ Integrate Total
Events 14 7 7 28
Attributes 40 14 22 76
Segments 19 (of which 7 sample/test to be deleted) 0 13 32

Key takeaways:

  1. Transactional layer (payments / bonuses / account) is essentially complete. 14 of 19 base event-triggered CIO campaigns are already running (Confirm Email, Deposit Successful/Failed/Pending, Withdrawal *, KYC, Bonus, FS, Tournament). This covers all FTD tracking, KYC tracking, and the payment flow.

  2. Backend already exposes most required attributes — just under different names. 14 “new attributes” from the playbook actually exist already (deposit_made → payment_deposit_succeeded, bonus_abuse_flag → segment ❗️Bonus Abuser Flag, vip_level → is_vip + player_loyalty_level, etc.).

  3. The real backend gap is 5 new events (player_session_ended, player_big_win, player_balance_low, player_wagering_completed, payment_withdrawal_cancelled) + 1 for Failed Deposit (payment_deposit_failed with error_type) + 1 CIO-only event (promo_code_redeemed).

  4. CIO-side work: ~13 new segments (suppression ❗️Dormant Archive, lifecycle Inactive 7/14/21/35d, churn risk, birthday tiers, Pre-VIP, KYC stalled).
