1. Обзор

Для проекта Ubuntu Quantum настроена отправка SMS через вебхук в Customer.io с доставкой через SMS-провайдера CM.com. Customer.io не имеет нативной интеграции с CM.com, поэтому используется блок «Send and Receive Data» (webhook action) внутри кампании.

Отправка SMS инициируется внешним HTTP-запросом на вебхук-URL кампании, после чего Customer.io формирует и отправляет запрос на CM.com API с данными из тригера.

1. Огляд

Для проєкту Ubuntu Quantum налаштовано відправку SMS через вебхук у Customer.io з доставкою через SMS-провайдера CM.com. Customer.io не має нативної інтеграції з CM.com, тому використовується блок «Send and Receive Data» (webhook action) всередині кампанії.

Відправка SMS ініціюється зовнішнім HTTP-запитом на вебхук-URL кампанії, після чого Customer.io формує і надсилає запит на CM.com API з даними з тригера.

1. Overview

For the Ubuntu Quantum project, SMS sending has been configured via a webhook in Customer.io with delivery through SMS provider CM.com. Customer.io does not have a native CM.com integration, so the "Send and Receive Data" block (webhook action) is used inside a campaign.

SMS sending is initiated by an external HTTP request to the campaign's webhook URL. Customer.io then constructs and sends a request to the CM.com API using data from the trigger.

4. Request Body to CM.com

Below is the JSON body configured in the "Send and Receive Data" block in Customer.io. Liquid variables {{trigger.*}} are populated from trigger data:

{
  "messages": {
    "msg": [
      {
        "allowedChannels": ["SMS"],
        "from": "0000",
        "to": [
          {
            "number": "{{trigger.phone | remove: '+' | remove: ' ' | remove: '-' | remove: '(' | remove: ')' | remove: '_'}}"
          }
        ],
        "minimumNumberOfMessageParts": 1,
        "maximumNumberOfMessageParts": 3,
        "body": {
          "type": "auto",
          "content": "{{trigger.message}}"
        },
        "reference": "cio-{{trigger.phone | remove: '+' | remove: ' ' | remove: '-' | remove: '(' | remove: ')' | remove: '_'}}-{{'now' | date: '%Y%m%d%H%M%S'}}"
      }
    ]
  }
}

Sender Name: имя отправителя PantherBet используется в конфигурации, но не подтверждено, зарегистрировано ли оно как Sender ID в CM.com. Необходимо верифицировать в CM.com Dashboard.
