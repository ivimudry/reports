1B - Payment Declined Flow (Weeks 1-2, parallel with Welcome)
This flow uses only the existing Deposit_canceled event - no new dev work needed. It can be built immediately.
What to build
1. Error-type routing: 6 error categories (Insufficient Funds, Wrong Card Details, Gateway Error, 3DS Failed, Crypto Failed, Deposit Limit). Each gets a unique in-app message and email. See strategy doc Section 4.1 for the full table.
2. Follow-up sequence: max 3-4 touches over 2 hours (down from current 5 emails + 3 SMS).
3. Anti-spam: if 2+ declines in 7 days, suppress further messages.
4. RG compliance: Deposit Limit Reached = zero promotional follow-up.

Phase 3: Retention & Reactivation (Weeks 5-8)
This is the highest-impact lifecycle. Most of Oshi’s audience is churned - this is where you win them back.
PREREQUISITES: inactivity_7d/14d/30d events, wagering_completed event, bonus_expired event, product_inactive event must be live.
3A - Active Retention Triggers (parallel campaigns)
These are NOT a single linear campaign. Each is a separate CIO campaign with segment filters:
• wagering_completed → reload offer (50% + 50 FS)
• balance_low → top-up pop-up (50 FS on next deposit)
• session_ended (30+ min, $0 balance) → email after 2hr (25% reload)
• 5 days since deposit, still logging in → Lootwheel/30 no-dep FS
Each campaign checks the player’s current lifecycle segment to show the right offer.
3B - Reactivation Tiers (4 tiers)
Build 4 separate CIO campaigns, one per tier. See strategy doc Section 10.2 for full offer sequences.
Tier 1 (7d inactive): 7 days, 3 touches, starts with 50 FS no deposit
Tier 2 (14d inactive): 10 days, 4 touches, 75 FS no deposit + Lootwheel
Tier 3 (30d inactive): 14 days, 4 touches, 100 FS no deposit + Lootbox
Tier 4 (60d+ dormant): 21 days, 3 touches, 150 FS no deposit + ultimate offer

Exit: any deposit at any tier → clear reactivation tags, return to Active Retention.
Tier 4 exhausted with no deposit → mark dormant_archive. Quarterly announcements only.

Rules That Apply to ALL Campaigns
Suppression Logic
• active_bonus = true → suppress ALL bonus offers in ALL flows
• bonus_abuse_flag = true → exclude from high-value reactivation offers
• Self-exclusion active → remove from ALL promotional flows, zero exceptions
• Deposit limit reached → no deposit incentives until reset

Send Time Optimization
Use playing_time_segment attribute when available. Until then:
• DE: 18:00-20:00 CET
• AU: 19:00-21:00 AEST
• CA: 18:00-20:00 ET/PT
• Welcome Flow Email #1 and in-app popups are immediate (not time-gated).
Tagging Convention
Use these CIO tags consistently across all flows:
• no_ftd - completed Welcome without depositing
• no_ftd_cooldown_30d - 30-day suppression after Welcome exhaustion
• FTD_only - has FTD but no 2nd deposit
• welcome_complete - completed all 4 welcome pack deposits
• welcome_incomplete_step_4 - completed 3/4 deposits
• reactivation_entry (event) - with attributes: tier (1-4), source (welcome/nutrition1/nutrition2/nutrition3/retention)
• dormant_archive - exhausted Tier 4 reactivation with no return
A/B Testing Protocol
Every new flow: 50/50 split vs current flow (or vs no flow for new lifecycles). Run 2+ weeks minimum. Track: conversion rate (deposit), revenue per player, unsubscribe rate, bonus cost per conversion. Report results to Farkhad before full deployment.
